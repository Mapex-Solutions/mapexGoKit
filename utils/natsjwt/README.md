# natsjwt — Issue and parse NATS user JWTs (Ed25519/nkey)

Issues and parses **NATS user JWTs** — the Ed25519/nkey-signed credentials NATS leaf nodes validate locally during decentralized auth. Wraps [`nats-io/jwt/v2`](https://github.com/nats-io/jwt) and [`nats-io/nkeys`](https://github.com/nats-io/nkeys) with a flat API that mirrors the shape of `utils/zerovalue` and `utils/random`.

> Package name: `natsjwt` (directory: `natsjwt/`).

Typical flow:

- **Issue** during device provisioning (with the platform's signing seed).
- **Parse** in HTTP middleware that verifies a device-presented bearer token.

## Surface

```go
type UserClaims struct {
    Name        string             // NATS "name" claim — Mapex puts the asset uuid here
    Issuer      string             // public key that signed the JWT (read-only on parse)
    BearerToken bool               // true → leaf accepts JWT without nonce challenge (IoT path)
    Tags        map[string]string  // flat string map (orgId/assetUUID/templateId, …)
    Pub         PermissionList     // subjects the device may publish on
    Sub         PermissionList     // subjects the device may subscribe to
}

type PermissionList struct {
    Allow []string
    Deny  []string
}

func IssueUserJWT(signingKeySeed string, userPubKey string, claims UserClaims, ttl time.Duration) (jwt string, jti string, expiresAt time.Time, err error)
func ParseUserJWT(jwtToken string)                                                        (UserClaims, string, time.Time, error)
```

### `IssueUserJWT`

| Argument | Meaning |
|---|---|
| `signingKeySeed` | Account NKey seed (or a delegated signing key listed in the account JWT). Empty → `"natsjwt: signing key seed is empty"`. |
| `userPubKey` | Public half of a fresh user keypair generated by the caller. The device that holds the matching seed connects as that user. Empty → `"natsjwt: user public key is empty"`. |
| `claims` | The fields documented above. |
| `ttl` | Added to `time.Now()` to compute the expiry. The expiry is **truncated to second precision** so the returned `expiresAt` matches what `ParseUserJWT` will read back (NATS "exp" is Unix-seconds). |

Returns:

- `jwt` — the signed JWT string.
- `jti` — the JWT ID assigned by `nats-io/jwt` (a **content-derived hash, not a random UUID** — the library computes JTI from the claim hash for tamper-evidence). Persist when you need to revoke a JWT before its natural expiry.
- `expiresAt` — UTC, second-truncated.

Tags from `claims.Tags` are emitted as `"key:value"` strings via `uc.Tags.Add(...)`. Tag keys and values are normalised to lowercase by the NATS library; encode case sensitivity in the value yourself (base64, separator, etc.) if you need it.

### `BearerToken` mode

When `claims.BearerToken == true`, the leaf accepts the JWT without challenging the device to sign a connection nonce with the user nkey seed. This is the **IoT path** — devices connect with `Username = devID` + `Password = JWT` and never need to hold the user nkey seed. Use for MQTT-protocol device JWTs.

### `ParseUserJWT`

Decodes a JWT and returns the claim subset. **Does NOT verify the signature against an issuer** — that is the leaf's job at MQTT CONNECT time and the HTTP middleware's job for refresh requests. Callers that need verification should layer it on top of this read.

Errors:

- `"natsjwt: jwt token is empty"` for empty input.
- `"natsjwt: decode user jwt: <err>"` for malformed JWTs.

Returns `(claims, jti, expiresAt, nil)` on success. Tags are split on the first `:` — values that contain a colon keep the suffix (no further escaping is performed).

## Usage

```go
// Generate user keypair (caller-side)
ukp, _ := nkeys.CreateUser()
userSeed, _ := ukp.Seed()
userPub, _  := ukp.PublicKey()

token, jti, exp, err := natsjwt.IssueUserJWT(
    accountSigningSeed,
    userPub,
    natsjwt.UserClaims{
        Name:        assetUUID,
        BearerToken: true, // MQTT device
        Tags:        map[string]string{"orgid": orgID, "asset": assetUUID},
        Pub:         natsjwt.PermissionList{Allow: []string{"telemetry." + assetUUID + ".>"}},
        Sub:         natsjwt.PermissionList{Allow: []string{"cmd." + assetUUID + ".>"}},
    },
    24*time.Hour,
)

// Persist (token, jti, exp) — the device receives `token`, the platform stores
// `jti` for revocation and `exp` for cleanup.

// Later, in middleware:
claims, parsedJTI, parsedExp, err := natsjwt.ParseUserJWT(token)
```

## Notes

- The library overwrites the `ID` (JTI) claim with a content-hash during `Encode`. The function decodes the freshly signed JWT to read the JTI back rather than predicting it.
- `Tags` parse uses the first `:` only — keys that themselves contain `:` are not supported.
- Pub/Sub allow-vs-deny semantics follow NATS rules: an empty `Allow` with empty `Deny` means "no permission" (broker rejects everything); a non-empty `Allow` with empty `Deny` means "exactly these subjects are allowed".
